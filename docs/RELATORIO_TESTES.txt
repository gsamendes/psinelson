Backend:

Observacao: os testes NAO dependem de banco de dados real (PostgreSQL/Neon).

(mocks) ou validacao de contrato/seguranca, e os demais ficam para o 8o periodo.

7. LIMITACOES E TESTES PENDENTES (8o PERIODO)
================================================================================
- Os testes nao acessam um PostgreSQL real. Os servicos de backend foram
  validados com Prisma mockado (contrato + seguranca + regras), nao persistencia.
- Login real ponta a ponta (frontend -> API -> banco) sera validado apos
  provisionar o banco no Neon e ligar VITE_USE_MOCKS=false.
- Testes de integracao com banco de teste e testes E2E (ex.: Playwright/Cypress)
  ficam para a proxima etapa.
- Validacao de campos obrigatorios na UI de login/cadastro ainda nao existe
  (campos com valores padrao) - nao testada para nao inventar funcionalidade.
- Prontuario clinico real NAO foi implementado (placeholder seguro). Nenhum teste
  cria ou le dado clinico real.
